This document describes how to run XD_XD (Kohei) 's solution for Spacenet Challenge Round 2 with using nvidia-docker.

Algorithm and Model Summary: My final submission (18th submission, 17.csv) during the online submission phase is an averaging ensemble of three end-to-end U-Net models. The best individual model uses the combination of multispectral images and OpenStreetMap masks as the input layers of the neural network model.

Directory Structure¶

My script on the docker container expects the following directory structure, which is the same one described on the official final testing guide.

/data
├── test
│   ├── AOI_2_Vegas_Test
│   ├── AOI_3_Paris_Test
│   ├── AOI_4_Shanghai_Test
│   └── AOI_5_Khartoum_Test
└── train
    ├── AOI_2_Vegas_Train
    ├── AOI_3_Paris_Train
    ├── AOI_4_Shanghai_Train
    └── AOI_5_Khartoum_Train

All working files for my solution will be stored on /data/working. For saving the training runtime, you can deploy working files and skip the training step. Those files are included in kohei-solution-20170612_4.zip. External resources (OSM shapefiles) are included in the docker image.

mul_bandcut<AOI_NAME>.csv percentile stats of multispectral images
rgb_bandcut<AOI_NAME>.csv percentile stats of multispectral images
<AOI_NAME>_val_weights.h5 A model weight of trained network model
<AOI_NAME>_val_evalhist_th.csv A result of parameter optimization on validation set

Please be aware of the remaining disk space. The size of Docker image grows up to 9.3 GB. When the disk space is empty during the training step, some working file will be broken.

macbook$ docker-machine scp kohei-solution-20170612_4.zip aws01:~/
macbook$ docker-machine ssh aws01

## This solution consumes huge amount of disk space.
## Mounting a large magnet volume (> 1 TB) is recommended when you run the instruction of training step from scratch.
ubuntu@aws01:~$ sudo mount /dev/xvdf /data

Build Docker Image¶

## 1. Extract my solution archive.
ubuntu@aws01:~$ unzip kohei-solution-20170612_4.zip
ubuntu@aws01:~$ cd kohei-solution

## 2. Build docker image (image name: kohei-sol)
ubuntu@aws01:~/kohei-solution$ sudo docker build -t kohei-sol .
(snip)
Successfully built 691ea5072dd3
Successfully tagged kohei-sol:latest

## 3. Run and exit a container (container name: kohei-container)
$ sudo nvidia-docker run \
    --name kohei-container \
    -v /data:/data \
    -ti kohei-sol \
    bash

root@xxxxxxxx:~# exit

Instructions for Inference Step (with using trained models)¶

For inference step, the preprocessed files and trained model weights must be stored on /data/working. This solution can skip the training step when the files of the minimum requirements described on the above are deployed on /data/working.

## 0. Copy working files including trained model weights into the container (kohei-container)
$ sudo docker cp working kohei-container:/data/

## 1. Start the docker container by nvidia-docker
$ sudo nvidia-docker start -a kohei-container -i

## 2. Inference
## (the prefix of directory name on the datapath must be one of AOI_2_Vegas, AOI_3_Paris, AOI_4_Shanghai and AOI_5_Khartoum.)
root@xxxxxxxx:~$ ./test.sh \
    /data/test/AOI_2_Vegas_Test \
    /data/test/AOI_3_Paris_Test \
    /data/test/AOI_4_Shanghai_Test \
    /data/test/AOI_5_Khartoum_Test \
    v17.csv

Instructions for Training Step from scratch¶

train.sh is the script to train models from scratch. Note that this script removes /data/working at first step.

## 1. Run Container (container name: kohei-container)
$ sudo nvidia-docker start -a kohei-container -i

## 2. Training
root@xxxxxxxx:~$ ./train.sh \
    /data/train/AOI_2_Vegas_Train \
    /data/train/AOI_3_Paris_Train \
    /data/train/AOI_4_Shanghai_Train \
    /data/train/AOI_5_Khartoum_Train

Appendix: Time Cost Analysis¶

p2.xlarge is 2 times slower than a personal PC with GeForce 1080. It's amazing how quickly GPU is evolving! Tesla K80 on p2.xlarge was released in 2014. My 17th submission (best one. v18.csv. not final) requires 6 models and scores over 70k on provisonal phase. However, it may exceed the time limitation described on the final test guide. This is the reason why I choosed 18th submission (ensemble of 3 models, v17.csv) as final one.
